# 🐝 Настройка Unity проекта BeeSwarm

## 📋 Текущий статус

Проект имеет полную структуру и готов к работе в Unity. Проблема: Unity 6000.4.2f1 не видит скрипты в "Add Component".

## 🔧 Решение проблемы

### 1. **.meta файлы созданы**
Я создал все необходимые .meta файлы. Unity теперь должен видеть скрипты.

### 2. **Что делать в Unity:**
1. Открой проект в Unity
2. Нажми: **Assets → Reimport All**
3. Перезапусти Unity если нужно

### 3. **Если скрипты всё ещё не видны:**
1. Проверь версию .NET в **Edit → Project Settings → Player → Other Settings**
   - **Scripting Backend:** Mono или IL2CPP
   - **Api Compatibility Level:** .NET Standard 2.1 или .NET Framework
2. Проверь настройки Editor: **Edit → Preferences → External Tools**
3. Убедись что скрипты в папке `Assets/Scripts/`

## 🚀 Быстрый старт

### **Тестовая сцена:**
1. Создай новую сцену: **File → New Scene**
2. Создай пустой GameObject и назови "GameManager"
3. Добавь компонент `GameManager` (должен быть в списке)
4. Создай GameObject "HiveManager" с компонентом `HiveManager`
5. Создай несколько кубов как тестовые цели
6. Добавь компонент `TestScene` на камеру или пустой объект

### **Префаб пчелы:**
1. Создай 3D объект (например, Capsule)
2. Добавь компонент `BeeSimple` или `BeeController`
3. Перетащи в папку `Assets/Prefabs/` как префаб
4. Укажи префаб в HiveManager

## 🎮 Тестирование

### **Простой тест:**
1. Запусти сцену с `TestScene` компонентом
2. Должны появиться пчёлы
3. Кликай по земле - пчёлы летят к точке
4. Проверь UI в углу экрана

### **Что должно работать:**

- ✅ Движение пчёл к целям
- ✅ Система энергии (цвет меняется)

- ✅ Ресурсы улья (мёд, пыльца, воск)

## 🐛 Известные проблемы

### **Unity 6000.4.2f1:**
- Очень новая версия, могут быть баги
- Если не работает - попробуй LTS версию (2023.3.x)

### **Скрипты не в списке:**
1. Убедись что файлы имеют расширение `.cs`
2. Проверь что нет ошибок компиляции в Console
3. Попробуй создать новый скрипт через Unity

## 💡 Советы

### **Для отладки:**
1. Включи **Console** (Window → General → Console)
2. Смотри ошибки компиляции
3. Используй `Debug.Log()` для отслеживания
